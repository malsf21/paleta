Overview

Welcome to the style guide for the World Affairs Conference! On our Paleta site, you'll find information to help you understand our design style: one that's based on simplicity and minimalism, even though both of those are heavily used graphic-design buzzwords.

In this style guide, you'll find quite a few things: a set of pages that explain how our design style works, files and examples for our design styling, and some occasional tips and tricks regarding how to make cool stuff for or about WAC. This style guide will be the most useful to our developers and designers, though all are welcome to use this as a resource. While we hope most of this is self-explanatory, please let us know (for now, that's just Matthew Wang), and we'll do our best to make sure this all makes sense.

Design Setup

There is no correct design setup, and this section isn't going to tell you what the "correct" style is. Rather, we're just going to outline what we recommend for design. Also, as a side note: our software assumes that you're using a Mac, since all of WAC is run on them. If you don't have a Mac, there are Windows versions or alternatives to all of these programs.

Note: Software marked with a star (*) are provided with education licensing. Ask your WAC supervisor for more details.
Software Name Purpose Link
Adobe Illustrator* Vector creation (e.g. logos, icons, illustrations)
Adobe Photoshop* Image manipulation (e.g. photo editing, posters, basic design)
Adobe InDesign* Layout editing and proceduralized generation (e.g. publication package editing, generating nametags)
Sip Mac colour picker (useful for coulour palettes)
Google Nik Collection Advanced image editing plugin (hooks into Photoshop) (e.g. photo filters)
Svgsus Mac SVG manipulation (mostly used for web design work)

All of the software works with WAC right out of the box; the only post-download setup step that's required is installing Nik's Photoshop plugin. Once you have that done, you're ready to start designing with WAC!

Projects

While this isn't technically needed in a style guide, here's a list of what kinds of projects that WAC designs throughout a single-year lifecycle, and an example (if possible).

Name Description Example
Delegate Package A package that delegates receive on the day of the conference. It contains information about the day's speakers, the schedule, and a staff list. This is typically designed with Adobe InDesign.
Registration Package A package that faculty advisers receive as registration is opened. It contains information about the general conference, as well as a preview of the scheduled content. This is typically made with Adobe InDesign.
Program Logos A set of logos that are used to advertise our programming, and are used in program posters, publication packages, and social media. These are typically made with Adobe Illustrator.
Program Posters A set of posters that advertise the conference programming (keynote, panels, and plenaries). They are placed around the school (UCC) and used online. These are typically made with Adobe Illustrator.
Recruitment Posters A set of posters that advertise conference staff positions throughout UCC and BH. This was made with Adobe Photoshop.
Nametags A set of nametags for conference staff, delegates, and faculty advisers. These are typically made with Adobe InDesign (and the data merge function).
Social Media Uploads A set of photos that are used on our Facebook, Instagram, and Twitter pages. These are typically made with Adobe Photoshop.
